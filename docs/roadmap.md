# Idle Brick Breaker — 발전 계획 (Roadmap)

토이로 시작하지만, 설계 단계에서 남겨둔 이음새(저장 계층 인터페이스, 프레임워크 독립 엔진, 앱 레이어 분리)를 따라 단계적으로 키운다. 각 단계는 이전 단계를 뒤엎지 않고 얹는다.

## Phase 0 — MVP (현재 목표)

- 방치형 벽돌깨기 코어: 자동 공, 탭 타격, 3종 업그레이드, 스테이지 진행.
- Vanilla JS + Canvas, `node:test` 단위 테스트.
- GitHub Pages 배포 + 티스토리 iframe 임베드.
- localStorage 저장 (+ 메모리 폴백).

**완료 기준:** 블로그에서 플레이 가능, 새로고침 후 진행 유지, 순수 로직 테스트 그린.

## Phase 1 — 게임성 강화 (재미 붙이기)

- **밸런스 튜닝**: 초반 텐션 곡선 조정(첫 업그레이드까지 시간), 성장 상수 재조정.
- **벽돌 커버리지 개선**: 실측 결과 공 궤적이 준주기적이라 일부 벽돌이 사각지대로 남아 스테이지 완전 클리어가 느림. 벽돌 충돌 시 미세 각도 지터(deterministic 유지) 또는 공 각도 다양화로 커버리지를 ergodic하게 개선. (현재는 다중 공 + 탭 수동 처리로 완화)
- **연출**: 벽돌 파괴 파티클, 탭 피드백, 콤보 표시, 사운드(옵션 토글).
- **공 다양성**: 관통/분열/폭발 공 등 특수 공 (업그레이드 트리 확장).
- **오프라인 수익**: `updatedAt` 기반으로 마지막 접속 후 경과 시간만큼 골드 정산 (SaveData에 이미 필드 존재 → 계산 로직만 추가).
- **접근성**: 색약 대응 팔레트, 키보드 조작(데스크탑), prefers-reduced-motion 존중.

**이음새 활용:** 특수 공/업그레이드는 `Config`·`Economy`·`Physics` 순수 모듈 확장으로 흡수. 저장 스키마 변경 시 `SAVE_VERSION` 올리고 `parseSave`에 마이그레이션 분기.

## Phase 2 — 서버 기록 (Supabase)

- **`SupabasePersistence` 구현**: 설계 4장의 `Persistence` 인터페이스(`load`/`save`)를 그대로 구현. 게임 코드 무변경.
- **동기화 전략**: 로컬 vs 서버 `updatedAt` 비교로 최신 채택(last-write-wins), 추후 병합 규칙 고도화.
- **인증**: 익명 → 소셜 로그인(선택). 익명 세션으로도 기록 저장 가능하게.
- **랭킹/리더보드**: `stats`(totalBricksBroken, totalGold, playTimeSec, maxStage) 기반 순위. 읽기 전용 뷰부터.
- **RLS**: Supabase Row Level Security로 본인 기록만 수정 가능하게.

**전제:** SaveData가 이미 서버 전송용 최소 페이로드(version/updatedAt 포함)라 스키마 재설계 불필요.

## Phase 3 — Next.js 이식 + 앱 레이어

- **엔진 이식**: `src/` 순수 모듈 + 어댑터를 Next.js 프로젝트로 이동. 엔진 무변경, Render/Input을 React 컴포넌트로 감쌈(설계 5장 어댑터 표 그대로).
- **라우팅/메타**: 공유용 OG 이미지, 개별 기록 공유 링크.
- **서버 라우트**: 랭킹 집계·검증을 서버에서(치팅 방지 1차).
- **번들 전략**: 게임 캔버스는 클라이언트 전용(`dynamic import`, `ssr:false`).

**이식 비용 최소화 원칙:** 티스토리 임베드는 계속 iframe(배포 URL) 유지 가능 → 블로그는 손 안 대고 백엔드만 성장.

## Phase 4 — 수익화 (선택)

- **상품 구조**: 광고 제거, 프리미엄 스킨/테마, 부스터(인게임 골드와 분리된 실제 결제).
- **결제**: Next.js 서버 라우트 + 결제 PG(예: Stripe/토스페이먼츠) + 영수증 검증. 구매 결과는 결국 `SaveData` 값 변경으로 귀결(설계 5장).
- **치팅 방지**: 결제·랭킹 관련 상태는 서버 신뢰(클라 계산 금지).
- **윤리 가드레일**: 페이투윈 지양, 확률형 아이템 지양. 방치형 재미를 해치지 않는 코스메틱 중심.

## 기술 부채 / 승격 체크포인트

- **테스트 승격**: 모듈이 커지면 `node:test` → Vitest(watch·커버리지). 어댑터도 jsdom/Playwright로 스모크.
- **Node 버전**: Node 20 LTS 기준(`.nvmrc`로 고정). CI 도입 시 동일 버전 사용.
- **빌드 도입 시점**: 단일 파일 배포가 필요해지면(CodeSandbox/오프라인) 의존성 없는 인라인 빌드 스크립트 추가.
- **타입 안전성**: 로직 복잡도가 올라가면 JSDoc 타입 → 필요 시 TypeScript 전환.

## 원칙

- YAGNI: 각 Phase는 실제 필요(재미·기록·수익)가 확인될 때 착수.
- 이음새 우선: 다음 Phase를 막지 않는 인터페이스만 미리, 구현은 미룬다.
- 엔진 순수성 사수: 게임 규칙은 DOM/네트워크/결제를 절대 모른다.
